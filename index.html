Why this is different from cloud AI tools

Most AI documentation tools (DAX, Nuance, Abridge) stream your audio to remote servers for processing. Present uses OpenAI Whisper compiled to WebAssembly, running the entire speech-to-text model inside your browser's sandboxed JavaScript engine. The AI note generator (Qwen3) also runs entirely on your GPU via WebGPU. No patient identifiers, audio, or notes ever leave the device.

Verify it yourself — in 60 seconds

  1. Open Chrome DevTools (⌘⌥I on Mac, F12 on Windows)
  2. Click the Network tab, then check Preserve log
  3. Record a voice note and click Generate Notes
  4. Filter by XHR / Fetch — you will see zero outbound requests from this app during transcription or generation
  5. The only network calls are the one-time model downloads from HuggingFace CDN on first load (model weights only — no patient data)

Model sources (open source, auditable)

  • Speech-to-text: onnx-community/whisper-base.en via Transformers.js — ~145 MB, English, cached after first load
  • Note generation: Qwen3-4B (4-bit quantized) via WebLLM — ~2.3 GB, cached after first load
  • App source code: github.com/pedscoffee/ideal-guacamole
